# QUICK ACTIONS FOR STRESSED USERS: GROUNDING & EXPECTATIONS

## Understanding the User State

When a stressed user clicks these quick actions, they are **not** in a calm, reflective state. They are likely:
- Overwhelmed and seeking immediate relief
- Experiencing fight/flight/freeze response
- Cognitively taxed (executive function offline)
- Looking for something that works **right now**
- Possibly panicking, anxious, or shutting down

**Design Principle:** For stressed users, every click must provide immediate value. No friction, no choices, no learning curve.

---

## 1. "5 MINS BREATHING" BUTTON

### User's Mental Model

When a stressed user clicks "5 Mins Breathing," they expect:
- Immediate calming guidance
- Someone to tell them what to do
- A timer they don't have to watch
- To feel different/better in 5 minutes

**What They're NOT Thinking:**
- "I wonder what breathing technique this is"
- "Let me read instructions first"
- "Should I choose box breathing or diaphragmatic?"

### Expected Destination

**Primary Destination:** **Guided Breathing Session - Immediate Start**

**Screen Should Show:**
- Animated breathing visual (expanding/contracting circle)
- Simple text: "Breathe in... 2... 3... 4..."
- No other options, no menus, no settings
- Progress indicator (subtle, not demanding attention)
- "Stop" button easily accessible if they need to exit

### Full User Journey for Stressed User:

**Click → Breathe → Feel Better → Done**

1. **Click** "5 Mins Breathing"
2. **Instant transition** (under 1 second) to breathing screen
3. **Animation starts immediately** with first inhale prompt
4. **Voice guidance optional** (if phone not on silent)
5. **Countdown visible** but not demanding focus
6. **Haptic feedback** on inhale/exhale switches (if device supports)
7. **Completion:** Gentle chime, "You did it" message
8. **Option:** Rate how you feel now vs. before (simple 3-face scale)
9. **Return** to previous screen or offer "Another 5 minutes?"

### What Makes This Work for Stressed Users:

| Design Element | Why It Matters |
|----------------|----------------|
| No setup screen | Can't make decisions when stressed |
| Starts immediately | Catches them before they change their mind |
| Visual focus | Gives overwhelmed brain something simple to track |
| Haptic feedback | Grounding through physical sensation |
| No skill required | Anyone can follow a breathing ball |
| Clear endpoint | "I only have to do this for 5 minutes" |

### Edge Cases:

| Situation | Expected Handling |
|-----------|-------------------|
| User clicks but isn't ready | "Start when you're ready" with pause button |
| User starts crying during breathing | Session continues, no judgment, tissues emoji optional |
| 5 minutes feels too long | Offer 2-minute option after first use |
| User falls asleep | Gentle wake-up at end? Or let them rest? |
| Interrupted mid-session | Auto-save progress? Option to resume? |

### Post-Breathing Reflection:

Simple check-in after completion:
- "How do you feel compared to before?"
- Options: 😌 Better, 😐 Same, 😟 Worse
- If "Better": "Want to try a quick grounding exercise?"
- If "Same/Worse": "That's okay. Sometimes it takes more than one try."

## 2. "MOOD CHECK" BUTTON

### User's Mental Model

When a stressed user clicks "Mood Check," they expect:
- A quick way to name what they're feeling
- Validation that their feelings are okay
- Possibly next steps based on what they share
- Minimal effort (they can barely think right now)

**What They're NOT Thinking:**
- "Let me write a paragraph about my emotional state"
- "I should rate 15 different dimensions of affect"
- "This will be useful data for later analysis"

### Expected Destination

**Primary Destination:** **Ultra-Simple Mood Check-in Screen**

**Screen Should Show:**
- Large, friendly faces with simple labels
- Maximum 5-6 options (cognitive load matters)
- One-tap selection
- Immediate feedback/validation

### Mood Options for Stressed State:

| Face | Word | Color | What User Feels |
|------|------|-------|-----------------|
| 😫 | Overwhelmed | Purple | "Too much, can't cope" |
| 😠 | Irritable | Red | "Everything's annoying me" |
| 😰 | Anxious | Orange | "Heart racing, can't settle" |
| 😔 | Down | Blue | "Sad, heavy, low" |
| 🥱 | Exhausted | Gray | "No energy, depleted" |
| 😐 | Numb | Light Gray | "Can't feel anything" |

### Full User Journey for Stressed User:

**Click → Tap → Validate → (Optional) Act**

1. **Click** "Mood Check"
2. **See simple mood faces** (large, tappable)
3. **Tap one** that matches current state
4. **Immediate validation:** "It's okay to feel [emotion]"
5. **Brief explanation:** "[Emotion] often shows up as [body sensation/thought pattern]"
6. **Suggested next step:** "Would [recommended action] help right now?"
7. **Option to accept suggestion** or return to previous screen

### Validation Messages by Mood:

| Mood | Validation | Body Connection | Suggested Action |
|------|------------|-----------------|-------------------|
| Overwhelmed | "This is a lot to carry right now." | Shoulders tight, breath shallow | "Try 2 minutes of breathing?" |
| Irritable | "Your frustration makes sense." | Jaw clenched, fists tight | "Grounding might help release this." |
| Anxious | "Anxiety is trying to protect you." | Heart racing, butterflies | "Let's do a quick grounding exercise." |
| Down | "It's heavy right now. That's real." | Chest heavy, body slow | "Even 1 minute of self-compassion?" |
| Exhausted | "You've been running on empty." | Eyes heavy, body aching | "Rest is allowed. 5 minute breathing?" |
| Numb | "Sometimes we need to go numb to survive." | Feeling disconnected | "Grounding can help you feel present." |

### Data Collection (Secondary):

After mood selection, optionally:
- "How intense is this?" (1-5 slider)
- "Want to add a note?" (optional, not required)
- Log to diary automatically with timestamp

### If User Taps Multiple Moods:

Allow multi-select with message:
- "It's possible to feel more than one thing."
- "Right now you're feeling [X] and [Y]."
- "That's valid and human."

## 3. "GROUNDING" BUTTON

### User's Mental Model

When a stressed user clicks "Grounding," they expect:
- To feel more present and less floating/disconnected
- Something that engages their senses
- A quick way to "come back" to their body
- To interrupt spiraling thoughts

**What They're NOT Thinking:**
- "Which of the 27 grounding techniques should I choose?"
- "Let me read about the theory behind grounding first"
- "I should set up my grounding preferences"

### Expected Destination

**Primary Destination:** **Quick Grounding Exercise - 5-4-3-2-1 Technique**

**Why 5-4-3-2-1?**
- Most evidence-based for acute stress
- Engages all senses sequentially
- Gives overwhelmed brain a structured task
- Takes about 2-3 minutes
- Can be done anywhere, eyes open or closed

### Screen Should Show:

**Simple, progressive prompts:**

```
Grounding Exercise

🔍 LOOK: Name 5 things you can see
[Tap when you have 5]

👂 LISTEN: Name 4 things you can hear
[Tap when you have 4]

🫂 FEEL: Name 3 things you can physically feel
[Tap when you have 3]

👃 SMELL: Name 2 things you can smell
[Tap when you have 2]

👅 TASTE: Name 1 thing you can taste
[Tap when you have 1]

[Optional: Breathe] Take one deep breath
```

### Full User Journey for Stressed User:

**Click → Follow → Complete → Grounded**

1. **Click** "Grounding"
2. **See simple 5-4-3-2-1 interface**
3. **Progress through each sense** at their own pace
4. **No timers, no pressure** (they control the pace)
5. **Completion message:** "You're here. You're present. Well done."
6. **Option:** "How grounded do you feel now?" (1-5 scale)
7. **Option:** "Try breathing exercise?" or "Return home"

### Alternative: Quick Grounding Menu

If user might want choice, offer simple card options:

| Option | Duration | Best For |
|--------|----------|----------|
| 🌬️ 5-4-3-2-1 Senses | 2-3 min | Anxiety, panic, dissociation |
| 🪑 Chair Grounding | 1 min | Feeling floaty, disconnected |
| 💧 Temperature | 30 sec | Overwhelm, heat in body |
| 🌍 Environmental Scan | 2 min | Any stress state |

But for stressed users, **default to 5-4-3-2-1** with one tap.

### Grounding Variations by Context:

| User Situation | Adapted Grounding |
|----------------|-------------------|
| In public, can't speak | Text-only version, read silently |
| At home, can speak | Voice-guided option available |
| Eyes closed preferred | Audio-only mode |
| With squad | Option to ground together (shared experience) |

### Completion Validation:

After grounding, simple reflection:
- "Before grounding: [mood from mood check?]"
- "After grounding: [quick re-check]"
- "Your nervous system thanks you."

## 4. "DAILY JOURNAL" BUTTON

### User's Mental Model

When a stressed user clicks "Daily Journal," they expect:
- A safe place to dump their thoughts
- No judgment, no structure if they don't want it
- Possibly privacy (this is for them, not for sharing)
- To feel lighter after writing

**What They're NOT Thinking:**
- "Let me answer these 5 structured prompts"
- "I should write something profound for posterity"
- "I need to use proper grammar and spelling"

### Expected Destination

**Primary Destination:** **Simple, Blank Text Entry with Soft Prompts**

**Screen Should Show:**
- Large text area (ready to type or dictate)
- Gentle placeholder: "What's on your mind? No rules, no judgment."
- Minimal formatting options (maybe none)
- Save button prominent
- Privacy reminder: "This is just for you"

### Full User Journey for Stressed User:

**Click → Write/Dictate → Save → Release**

1. **Click** "Daily Journal"
2. **See blank canvas** with soft lighting
3. **Start typing or tap microphone** to dictate
4. **Write as much or as little** as they want (one word to pages)
5. **Save when done** (or auto-save as they write)
6. **Optional prompt:** "How do you feel after writing?"
7. **Entry saved** to private journal timeline

### For Users Who Want Guidance:

Offer a "Need a prompt?" expandable section with:

| Prompt Type | Examples |
|-------------|----------|
| Simple | "What's taking up space in your mind right now?" |
| Feeling-focused | "What emotion is loudest right now?" |
| Body-focused | "Where in your body do you feel the stress?" |
| Gratitude-light | "Is there one small thing that's okay right now?" |
| Unfiltered | "Write whatever comes. No editing allowed." |

### Voice-to-Text Integration:

Critical for stressed users who:
- Can't type (hands shaking)
- Need to close eyes
- Process better by speaking
- Are in a position where typing is hard

**Voice Experience:**
- Tap microphone icon
- Start speaking
- Real-time transcription
- Edit after if desired

### Post-Journal Release:

After saving, consider offering:
- **"Done for now"** → Return to main screen
- **"Read back"** → Sometimes hearing yourself helps
- **"Identify one thing"** → "What's one small need right now?"
- **"Burn option"** → For users who want to release without saving (privacy/security)

### Privacy Anchors:

Stressed users worry about:
- "Will someone read this?"
- "Is this stored forever?"
- "What if I die and someone finds it?"

Address visibly:
- "Your journal is private. Stored only on your device." (if true)
- "Password protected. Face ID required." (if applicable)
- "Delete any entry anytime."

## INTEGRATION: HOW QUICK ACTIONS WORK TOGETHER

### Suggested Flow for Stressed User:

A user might naturally move through multiple quick actions:

```
Feeling overwhelmed
         ↓
   [MOOD CHECK]
   "I'm overwhelmed"
         ↓
   "Try grounding?" 
         ↓
   [GROUNDING]
   5-4-3-2-1 exercise
         ↓
   "How about 5 minutes of breathing?"
         ↓
   [5 MINS BREATHING]
   Guided session
         ↓
   "Want to write about how that was?"
         ↓
   [DAILY JOURNAL]
   Quick entry
         ↓
   Feel more regulated
```

### Quick Action Grid for Stressed States:

| Current State | Best First Action | Second Action | Avoid |
|---------------|-------------------|---------------|-------|
| Panicking, can't breathe | Breathing (immediate) | Grounding (after breath) | Journal (too much cognition) |
| Dissociating, numb | Grounding (5-4-3-2-1) | Mood Check | Breathing (may increase floatiness) |
| Raging, irritable | Grounding (physical) | Journal (vent) | Breathing (may feel constricting) |
| Spiraling thoughts | Grounding (sensory) | Breathing | Journal (may reinforce spiral) |
| Heavy, sad, low | Mood Check | Journal | Breathing (may feel like effort) |
| Exhausted, empty | Breathing (restful) | Mood Check | Grounding (may feel like work) |
| Mixed, confused | Mood Check | Any | None (all may help) |

## DESIGN SPECIFICS FOR STRESSED USERS

### Visual Design Principles

| Principle | Implementation |
|-----------|----------------|
| Low cognitive load | One thing per screen, no choices when possible |
| High contrast | Easy to see through tears or blurred vision |
| Soft colors | No bright alerts or aggressive colors |
| Large targets | Fat-finger friendly (hands shaking) |
| Progress visible | "You're 2 of 5 minutes done" not "67% complete" |
| No scrolling | If possible, everything above the fold |

### Interaction Design

| Interaction | Design for Stress |
|-------------|-------------------|
| Buttons | Minimum 44px height, plenty of padding |
| Back navigation | Prominent, always available |
| Accidental taps | Undo option, confirm destructive actions |
| Timeouts | No auto-logout during exercises |
| Interruptions | Save state, easy resume |

### Tone of Voice

| Situation | Language |
|-----------|----------|
| Instructions | Simple, direct: "Breathe in. Hold. Breathe out." |
| Validation | Warm, accepting: "This is hard. You're doing it." |
| Completion | Celebratory but calm: "You did it. That matters." |
| Check-ins | Gentle: "How's that feeling now?" |
| Suggestions | Optional: "Would you like to try...?" |

### What NOT to Do When User is Stressed:

- ❌ Ask for ratings before helping ("Rate your anxiety 1-10" → NO)
- ❌ Show them data/graphs
- ❌ Require account actions (login, upgrade, etc.)
- ❌ Have them make choices (which exercise? how long?)
- ❌ Use triggering language ("calm down," "relax," "it's not that bad")
- ❌ Assume they remember anything (no "as we discussed earlier")
- ❌ Hide the exit (always let them leave)

## COMPARISON: STRESSED VS. CALM USER EXPERIENCE

| Element | Stressed User Needs | Calm User Might Want |
|---------|---------------------|----------------------|
| Breathing | Immediate start, guided | Choose technique, set duration |
| Mood Check | Simple faces, one tap | Detailed scale, multiple dimensions |
| Grounding | 5-4-3-2-1 default | Choose from 10+ techniques |
| Journal | Blank page, dictate option | Prompts, templates, reflection questions |
| Time perception | "Just get through this" | "I have time to explore" |
| Goal | Relief now | Insight, growth, tracking |

## ERROR STATES & STRESSED USERS

### What Happens When Things Go Wrong?

| Error | Stressed User Reaction | Design Response |
|-------|----------------------|------------------|
| App crashes mid-exercise | "See? Nothing works." | Auto-restart with apology: "We're sorry. Want to continue where you left off?" |
| Can't find the button | "I'm too stupid for this." | Big buttons, consistent placement, back always available |
| Exercise feels wrong | "I'm doing it wrong." | "There's no wrong way to breathe/ground. Just being here helps." |
| No improvement after | "Even this didn't work." | "Sometimes it takes a few tries. Your body is doing its best." |
| Internet drops | "Now I'm really alone." | Offline mode, cached exercises, "You're still okay without internet." |

## MEASURING SUCCESS FOR STRESSED USERS

### Metrics That Matter

| Metric | What It Tells Us |
|--------|------------------|
| Completion rate | Did they stay through the whole exercise? |
| Time to first action | How quickly did we help them? |
| Return rate after stress | Did we help enough to come back? |
| Multi-tool usage | Did they need more than one intervention? |
| Post-exercise rating | Did they feel better? (1-3 simple scale) |
| Abandon rate | Where did we lose them? |

### Success Looks Like:

- User clicks "5 Mins Breathing" → Completes full session → Rates "better"
- User clicks "Mood Check" → Identifies emotion → Accepts suggested grounding
- User opens Journal → Writes/dictates for 2+ minutes → Saves entry
- User uses 2+ tools in one session → Reports feeling more regulated

## SUMMARY: QUICK ACTIONS FOR STRESSED USERS

| Action | User Wants | Gets Immediately | Feels After |
|--------|------------|------------------|-------------|
| **5 Mins Breathing** | To calm down | Guided breathing, no decisions | Regulated, centered |
| **Mood Check** | To name the feeling | Simple mood selection, validation | Understood, less alone |
| **Grounding** | To feel present | 5-4-3-2-1 exercise, sensory focus | Anchored, connected |
| **Daily Journal** | To release thoughts | Blank page or voice, no judgment | Lighter, expressed |

**The Golden Rule for Stressed Users:** Remove all friction. Validate immediately. Offer help without asking. Never make them think harder than they already are.
